# AI 选股研判规则 (AGENTS.md)

为了确保选股分析的客观性与一致性，本工作空间内所有的选股与持仓研判任务必须遵循以下规则：

## 1. 50分制穿透式评分系统
对任何推荐、分析或持仓中的股票，必须逐一对照 [mission.md](file:///Users/yuezh/AIStocks/mission.md) 中的四大核心选股逻辑及竞争格局定价权进行打分。每个维度满分为 10 分，总分为 50 分：

1. **AI 基建上游原材料 (AI Infrastructure Upstream Raw Materials)** — 权重：10分
   * 评估该标的产品是否属于算力、数据中心、电网等 AI 基建必不可少的底层物理媒介或稀缺原材料。
   * **【物理/关键重要性阶梯打分】**：
      * **9.0 - 10.0分 (Bottleneck级)**：独占性、无替代性的 AI 底层物理瓶颈耗材/量测设备（如 HBM 精密光学测试设备、先进封装抛光垫/液、高导热 TIM）。
      * **7.0 - 8.5分 (Critical级)**：高准入门槛的 AI 供应链关键原材料，但存在潜在替代路径或多方供货（如 LCP 高频传输树脂、高纯化学过滤器、配电配变高耐压变压配件）。
      * **5.0 - 6.5分 (General级)**：通用半导体特种气体/化学品或电网通用连接配件，AI 建设带来实质性需求增长但非绝对垄断。
      * **< 5.0分 (Non-AI级)**：非 AI 基建相关产品。
   * **【AI 业务营收纯度调节因子 (AI Revenue Purity Adjustment Factor)】**（越纯得分越高）：
      * **高纯度 (AI/半导体相关主营业务营收占比 $\ge$ 60% 或属于纯 AI 物理瓶颈标的，如 ONTO, NVMI, FORM, ENTG)**：不扣分。若属于绝对壁垒垄断龙头，可额外加成 **+0.5分**（分项满分 10分为限）。
      * **中纯度 (20% $\le$ AI/半导体相关业务营收占比 < 60%，如 CE, MOD)**：标的虽然是 AI 供应链的关键一环，但公司整体业绩仍有相当比重受传统工业、周期或其他非 AI 业务稀释。该分项得分需扣减 **0.5 - 1.0分**。
      * **低纯度 (AI/半导体相关业务营收占比 < 20%，如 DD)**：公司为大型多元化综合性集团，AI 物理媒介仅占其极小的营收份额，整体业绩对 AI 景气度弹性极低。该分项得分强制扣减 **1.5 - 2.0分**，若扣减后该分项得分低于 7.0分，则直接触发淘汰机制。

2. **深厚护城河 (Deep Moat)** — 权重：10分
   * 评估技术壁垒、独占性、市场垄断格局、客户转换成本及行业准入资本/合规壁垒。
   * **【技术路径替代压力测试】**：若标的核心产品存在 2 年内被下游主流厂商（如台积电、英伟达等）新路线替代的明确可能（如玻璃基板替代有机载板、光电共封装技术突变等），本项得分最高不得超过 **6分**。
   * **【环保与法律负债压力测试】**：若企业面临重大潜在诉讼或合规支出，但该负债已被股价充分计入（表现为远期 PE < 12x 且自由现金流仍为正数），本分项不得过度扣分，应如实评估其业务技术垄断力；仅当债务风险对企业经营现金流构成实质性中断/破产威胁时，本项得分最高不得超过 **6分**。

3. **主升浪未完全启动 (Main Upward Wave Not Fully Started)** — 权重：10分
   * 评估当前估值与技术面走势。根据标的属性，分别采用**双轨制 (Dual-Path) 估值评估模型**，并引入技术形态调节因子：
   * **【路径 A：科技与装备成长股 (ONTO, KEYS, ENTG, Q, POWL, HUBB, ETN)】**：
      * 采用 **Forward PEG** 进行校准。
      * **9.0 - 10.0分**：`PEG <= 1.2`（估值明显低估，高增长未被充分定价）。
      * **7.0 - 8.5分**：`1.2 < PEG <= 1.6`（估值合理，高增长能有效匹配）。
      * **5.0 - 6.5分**：`1.6 < PEG <= 2.2`（估值偏高，适合持有，限制新开仓）。
      * **4.0 - 4.5分**：`2.2 < PEG <= 2.5`（估值透支，触发分批止盈/减仓）。
      * **< 4.0分**：`PEG > 2.5`（估值泡沫化严重，强力建议止盈/清仓）。
   * **【路径 B：传统材料、资源与公用事业/周期股 (CE, CC, DD, Dow)】**：
      * 采用 **Forward PE** 与 **PB** 双重指标进行估值校准。
      * **9.0 - 10.0分**：`Forward PE <= 12x` 且 `PB <= 1.8x`（估值处于历史绝对低位，安全性极高）。
      * **7.0 - 8.5分**：`12x < Forward PE <= 18x` 或 `1.8x < PB <= 2.8x`（估值合理，反映了底部的温和复苏）。
      * **5.0 - 6.5分**：`18x < Forward PE <= 25x` 或 `2.8x < PB <= 4.0x`（估值温和偏高，受周期上行预期支撑，适合持有）。
      * **< 5.0分**：`Forward PE > 25x` 或 `PB > 4.0x`（估值严重透支，除非符合底部复苏特例，否则触发减仓/淘汰）。
   * **【技术形态与偏离度调节因子（适用于路径 A & B）】**：
      * **乖离率惩罚**：若股价偏离 50 日均线 (SMA 50) $\ge$ 20%，说明短期严重超买，“主升浪”得分强制扣减 **1.5分**。
      * **周期底部起跑例外条款**：若 Forward PE 较高但由于周期底部盈利极低，且其 `PEG <= 0.6` 且 `PB <= 2.5`（证明是周期反转起步期，盈利高速复苏），可豁免 PE 惩罚，主升浪得分可评为 **7.0 - 8.5分**。

4. **机构建仓痕迹明显 (Institutional Accumulation)** — 权重：10分
   * 评估大资金与 Smart Money 的持股集中度以及中长期技术图表形态。
   * **【机构持仓与技术形态打分】**：
      * **9.0 - 10.0分 (Strong Accumulation)**：机构持股比例 $\ge$ 80% 且近期有显著加仓，或者周/月 K 线呈现完美的收缩波动形态（如 VCP、杯柄底、圆底），上涨放量、回调缩量极其明显，处于爆发前夕。
      * **7.0 - 8.5分 (Healthy Ownership)**：机构持股比例 50% - 80%，或者技术形态处于中线震荡整理筑底阶段，量价关系健康，大资金平稳沉淀。
      * **5.0 - 6.5分 (Neutral/Weak)**：机构持股比例 < 50% 且无明显大资金流入，或者技术形态处于无序震荡或温和下行整理中。
      * **< 5.0分 (Distribution)**：机构持续流出/大额减持，或者股价处于高位且成交量异常放大但价格滞涨（显示筹码正在派发）。

5. **定价权与免于价格战压力的能力 (Pricing Power & Freedom from Price War Pressure)** — 权重：10分
   * 评估标的在其细分市场中的垄断定价权、下游客户的供求关系紧缺度，以及面对同类竞品或潜在价格战时的防御免疫力。
   * **【定价权与价格战免疫力阶梯打分】**：
      * **9.0 - 10.0分 (Monopoly/Shortage级)**：在细分领域拥有绝对的技术垄断定价权（如 ONTO 独占量测机台），或者由于行业产能极度匮乏，下游客户争相签订排他性长期协议以抢夺产能（如 MOD 的 CDU 液冷温控）。基本不受任何价格战威胁，毛利率极高且极其稳定。
      * **7.0 - 8.5分 (Consumable/Sticky级)**：虽有少量同行竞争，但属于客户的高复购、高黏性、低单价“非敏感性”核心耗材（如 FORM 的探针卡、ENTG 的高纯过滤器），客户转换成本高，不易受降价拼单影响，毛利率维持健康。
      * **5.0 - 6.5分 (Co-opetition级)**：行业处于良性竞争状态，但存在一定的技术迭代或同业低价渗透风险（如 CE 的 LCP 特种树脂面临潜在联产扩建，或 NVT 的普通机架面临多方竞争），毛利率存在温和波动压力。
      * **< 5.0分 (Commodity/Price War级)**：下游客户高度强势且多元采购，同业竞争极其激烈，技术壁垒中等偏上但极易陷入代际价格战和代工毛利率挤压（如 COHR 等集成器件模块光纤厂商，易受 Hyperscaler 压价及同业拼价影响），毛利率波动剧烈。

## 2. 研判报告与交付物标准 (Research Report & Deliverables Standards)
* **报告时间戳**：报告开头必须标明本次研判报告输出的具体时间，格式为 `YYYY-MM-DD HH:MM`（具体到分钟）。
* **评分表**：每个受研判的股票都必须列出分项得分及总分，并附带简要的核心逻辑提炼。
* **淘汰机制**：任何总得分低于 38 分或在“AI上游原材料”维度得分低于 7 分的标的，原则上不予保留或建议调出观察列表。
* **明确的下一步操作 (Next Steps)**：研判报告最后必须包含一个独立的“后续执行步骤 (Next Steps)”章节，给出清晰、具体、可操作的交易执行指引（如平仓顺序、建仓/加仓步骤、未部署现金的管理与后续跟踪监控指标），以形成闭环。

## 3. 触发机制 (Trigger Mechanism)
* **当用户输入 `scan` 指令时**，启动**全美股市场扫描与新候选股挖掘流程**：
  1. **读取规则**：读取最新的 `mission.md` 和 `AGENTS.md`。
  2. **全局市场扫描与新候选股挖掘**：首先执行第 7 节定义的“全美股市场扫描与新候选股挖掘”流程，通过网络检索和行业透视，突破当前观察列表限制，主动发现隐藏的、符合 AI 物理瓶颈/原材料定位（“AI基建上游原材料”分项评分 $\ge$ 7.0分）的优质标的。
  3. **输出并记录**：创建或更新 `candidate.md`，记录所选择的标的。：以清晰的列表形式向用户呈现新挖掘出的候选股名单、其核心物理层技术卡位与简要的评估结论。

* **当用户输入 `research` 指令时**，启动**聚焦观察列表的持仓与评估研判流程**：
  1. **读取规则与获取范围**：根据的 `mission.md` 和 `AGENTS.md`的规则，获取  `candidate.md` 观察列表中所有标的（包含候选股）以及用户所有个人退休账户 (IRA) 的全部持仓信息与可用现金购买力。
  2. **全量最新数据获取**：针对上述研判范围内的标的，抓取最新的行情、基本面财务数据（P/E, P/B, PEG 等）、机构持股集中度、最新重要新闻、财报/业绩指引或公告以及分析师预期等。
  3. **50分制穿透式评估**：严格依照第 1 节的 50分制评分规则，对观察列表中所有个股及持仓个股进行深入打分和交叉研判（包含乖离率惩罚、调动缓冲阀以及非标事件与特殊风险监控等条款）。
  4. **调仓方案与报告输出**：出具符合第 2 节标准的交付物，包括精确的时间戳、分项评分表、以高估值止盈/高分替代为核心的调仓方案（建仓/加仓/减仓/换仓/维持建议与精确资金配比），以及清晰的下一步操作（Next Steps）， 写入`research_report.html`。
  5.**更新Robinhood AI UpStream watch list**： 调用Robinhood mcp 更新`AI UpStream` watch list。移除低于35分的标的，并添加新符合条件的标的。 排序：首先将所有持仓的个股排在前面，按照得分高低。其他没有持仓 of 个股，按得分从高到低排在后面。**【防倒序重要安全规则】**：由于 Robinhood API 的添加机制为“后进先出”（LIFO - Last In, First Out），新添加的股票会被置顶（prepend）。为了确保 Robinhood 客户端中从上到下的显示顺序为“持仓股（降序） ➡️ 非持仓股（降序）”，通过 API 同步写入时，**必须以期望顺序的【逆序】（即从得分最低的非持仓股开始，一直到得分最高的持仓股结束）依次添加/写入**。
  6. **更新本地 HTML 报告并同步至 GitHub**：根据最新评估数据，重新生成 `/Users/yuezh/AIStocks/research_report.html`，并自动运行终端命令 `git add research_report.html && git commit -m "Auto-update research report" && git push` 将该文件推送同步至远程 GitHub 仓库。

## 5. 换仓与止盈逻辑 (Reallocation & Profit Taking)
* **细分领域排他性持仓规则 (Niche Exclusivity Rule) - 【防重复持仓安全阀】**：
  * 为规避同行业/同技术路线的集中度风险，在 [mission.md](file:///Users/yuezh/AIStocks/mission.md) 定义的每个细分领域（如 LCP 材料、光学量测、数据中心液冷、探针卡测试等）中，**任何时候仅允许持有【一只】最优质（即 50 分制总分最高）的标的**。
  * **禁止重复持仓**：严禁在同一个细分领域内双重或多重持有标的（例如：禁止在已持有 LCP 基材/板材股 Rogers ROG 的情况下，继续持有 LCP 树脂原料股 Celanese CE，因为它们属于同一物理介质细分供应链的重复暴露）。
  * **换仓执行规则**：若同领域内的观察股/候选股总分升至 38 分以上，且与同领域内已有持仓股的**分差 $\ge$ 5 分**，则触发同领域内的**高分替代换仓**，必须清仓原有低分股，将资金全部换仓至该领域的新高分股。
* **止盈与再平衡优先级**：**估值降分止盈具有最高优先级**。若持仓股因估值透支被下调评分，必须执行对应的降分减仓/清仓操作。再平衡止盈仅适用于估值合理、因股价自然波动导致仓位溢出目标权重 50% 以上的健康上涨个股。
* **止盈触发机制**：
  * **主升浪分项得分评级下调止盈**：
    * 当持仓个股因大涨导致估值透支，其“主升浪未完全启动”得分降至 **4.0 - 4.5分** 时，触发**分批止盈（减仓 30% - 50% 目标仓位）**。
    * 当得分进一步降至 **3.5分及以下**（表明泡沫化严重）时，触发**全额止盈或仅保留微量底仓（不超过 5%）**。
  * **再平衡止盈**：在标的估值得分未降级时，当实际权重超出目标权重达 **50% 以上**，将溢出部分卖出回归目标权重。
* **换仓触发机制**：
  * **高分替代低分换仓**：当观察列表中出现新的优质标的，且其 50分制总得分显著高于当前持有的某只标的（**分差 $\ge$ 5 分**），且不违反细分领域排他性规则时，触发**换仓（Reallocate）**指令。
  * **闲置现金建仓例外条款**：若是使用闲置现金（可用购买力）对新晋升的标的进行首次建仓或加仓，**不受 5 分分差限制**，只要目标标的总分 $\ge$ 38 且**未持有该细分领域的其他标的**，即可直接部署。
  * **淘汰清仓换仓**：若个股总分跌破 **38分**，或“AI上游原材料”维度得分跌破 **7分**，直接触发淘汰机制，必须在下一次交易执行中完成清仓，所得资金优先配给分值最高且不冲突的标的或暂存为现金储备。
* **调动缓冲阀 (Hysteresis Buffer)**：
  * 为避免价格日常波动导致频繁换仓，估值降分（PEG/PE）需满足**连续 3 个交易日**均处于下调指，或伴随官方财报/业绩指引调整时，才正式调降主升浪得分并触发止盈。

## 6. 观察列表与候选股 (Watchlist & Candidates) 规则
* **候选股标准**：
  * 该标的符合 “AI 基建上游原材料” 核心定位（该项得分 $\ge$ 7分）。
  * 但目前由于估值偏高（“主升浪未完全启动”得分较低）或技术形态未整理好（“机构建仓”得分较低）导致总得分低于 38 分，不具备立即建仓条件。
* **准入与监控机制**：
  * 所有符合候选股标准的标的，必须自动通过 Robinhood API 添加至 `"AI Upstream"` 观察列表中。
  * 在定期运行 `research` 流程时，应对观察列表中的所有标的（包括未持有的候选股）进行全量数据抓取与 50分制重新评估打分。
  * **重要非标事件与特殊风险监控 (Notes & Risk Warnings)**：在执行研判时，必须检索并评估对观察池或持仓标的产生重大影响的非标事件与特殊风险（如重大诉讼、环保合规限制、核心供应链中断或地缘政治政策变化等），并在报告中列为“需要注意的notes”。针对面临重大不确定性诉讼的个股（如 Chemours CC），在其法律风险或财务前景实质明朗前，原则上维持谨慎观望或限制建仓。
* **晋升建仓机制**：
  * 当某只候选股由于股价回调、估值消化（“主升浪”得分上升）或机构买入痕迹明显（“机构建仓”得分上升）导致其 50分制总分升至 **38分及以上** 时，该股即自动“晋升”为**可建仓目标**。
  * 自动触发建仓建议，在下一期调仓方案中，优先使用闲置现金或通过削减低分持仓股的仓位，为其建立初始底仓（通常为 5% - 10% 目标权重）。
* **淘汰移除机制**：
  * 若标的由于基本面恶化（如核心产品技术路线被替代、“AI上游原材料”得分跌破 7分）或面临重大不可逆法律风险，必须通过 API 将其从观察列表中彻底移除。
* **排序机制 (Sorting Mechanism)**：
  * 每次更新 `"AI Upstream"` 观察列表后，必须按照 50分制评分总得分从高到低（降序）对观察列表内的标的进行排序并同步。**【防倒序重要安全规则】**：由于 Robinhood API 的添加机制为“后进先出”（LIFO），新添加的股票会被置顶。因此，在通过 API 进行列表同步或重新写入时，**必须按照期望排序的【逆序】（即得分最低的非持仓股最先写入，得分最高的持仓股最后写入）进行操作**，以保证用户在客户端看到的列表是从高到低的降序排列。

## 7. 市场扫描与新候选股挖掘机制 (Market Scanning & New Candidate Discovery)
* **目的**：主动打破观察列表的信息孤岛，防范由于观察列表静态化导致错失新兴 AI 供应链物理瓶颈/原材料龙头标的。
* **扫描与筛选步骤**：
  1. **定位匹配度测试**：根据公司主营业务、产品或技术路线，对照 `mission.md` 评估其是否为 AI 硬件、算力、数据中心或电力网络必不可少的底层物理媒介、稀缺原材料、良率提升关键耗材/设备或配电网络装备。
     * **概念检索**：使用 `search_web` 或 `mcp_yfinance_search` 对 `mission.md` 中的关键材料/设备概念进行检索（如 `CoWoS packaging materials stock` 等）。
     * **行业透视**：通过 `mcp_yfinance_get_industry_data` 或 `mcp_yfinance_get_sector_data` 遍历半导体、特种化学品、电气设备与科学仪器行业，发掘潜在的底层龙头。
  2. **50分制穿透式评估**：
     * 对检索出的每一个上游标的进行“AI基建上游原材料”分项评分评估。
     * 若其“AI基建上游原材料”分项得分 $\ge$ 7.0分，说明该股符合候选股定位，应当判定为**合格候选股 (Candidate)**，立刻通过 Robinhood API 自动将其添加至 `"AI Upstream"` 观察列表中。
     * 对新入选的候选股，在此后的 `research` 流程中进行全量打分和日常动态监测。
